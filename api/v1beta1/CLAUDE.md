# api/v1beta1

> Go type definitions for every CRD in the operator (group `neo4j.neo4j.com`, version
> `v1beta1`). These structs + their `+kubebuilder:` markers are the **source** for the
> generated CRD YAML and `zz_generated.deepcopy.go` — they are not generated themselves.

## Key files

| File | What it does |
|---|---|
| `groupversion_info.go` | Declares `GroupVersion` (`neo4j.neo4j.com/v1beta1`), `SchemeBuilder`, `AddToScheme`; carries `+groupName` / `+kubebuilder:object:generate=true`. |
| `neo4jenterprisecluster_types.go` | HA cluster CRD — largest file, ~40 validation markers (topology, TLS, plugins, monitoring, Aura fleet). |
| `neo4jenterprisestandalone_types.go` | Single-node CRD (dev/test). |
| `neo4jdatabase_types.go` | `Neo4jDatabase` — name, topology, Cypher version, CREATE options only. |
| `neo4jshardeddatabase_types.go` | `Neo4jShardedDatabase` (property sharding). |
| `neo4jbackup_types.go` | `Neo4jBackup` — Job-per-CR backup (`Target`, `Storage`, `Schedule`, chaining). |
| `neo4jrestore_types.go` | `Neo4jRestore`. |
| `neo4juser_types.go` / `neo4jrole_types.go` / `neo4jrolebinding_types.go` | Auth CRDs (privileges live on `Neo4jRole`). |
| `neo4jauthrule_types.go` | `Neo4jAuthRule`. |
| `neo4jplugin_types.go` | `Neo4jPlugin`. |
| `mcp_types.go` | Shared MCP config types (e.g. `MCPHTTPConfig`); no standalone CRD/Kind. |
| `seed_creds_target.go` | `SeedCredsTarget` interface methods on Cluster/Standalone (`GetExtraEnvFrom`, `SetExtraEnvFrom`, `TargetKindLabel`) — kept here so the api pkg need not import the controller pkg. |
| `types_test.go` | Plain `testing` unit tests for spec validation, defaults, deepcopy. |
| `zz_generated.deepcopy.go` | **GENERATED** (`// Code generated by controller-gen. DO NOT EDIT.`). Never hand-edit. |

## Key types & functions

- Eleven root Kinds, each with a `+kubebuilder:object:root=true` `<Kind>` + `<Kind>List` pair and an
  `init()` calling `SchemeBuilder.Register`: `Neo4jEnterpriseCluster`, `Neo4jEnterpriseStandalone`,
  `Neo4jDatabase`, `Neo4jShardedDatabase`, `Neo4jBackup`, `Neo4jRestore`, `Neo4jUser`, `Neo4jRole`,
  `Neo4jRoleBinding`, `Neo4jAuthRule`, `Neo4jPlugin`. Each has matching `<Kind>Spec` / `<Kind>Status`.
- `GroupVersion`, `SchemeBuilder`, `AddToScheme` (in `groupversion_info.go`).

## Conventions & gotchas

- **This is the SOURCE of generated artifacts.** Editing any struct or `+kubebuilder:` marker
  REQUIRES regenerating: `make manifests && make generate && make sync-all`. `make check-drift`
  (sync-all + bundle + `git diff --exit-code`) is a **blocking** CI gate — stale generated files
  fail merge. `make generate` rewrites `zz_generated.deepcopy.go`; never hand-edit it (it has the
  `DO NOT EDIT` header).
- **New fields need json tags** ("NOTE: json tags are required" scaffolding comment), and a new
  Kind needs the `root=true` Kind+List pair plus a `SchemeBuilder.Register` in `init()`.
- **No `edition` field** — the operator always assumes Enterprise (Invariant 3); grepping `edition`
  in this dir returns nothing. Don't add one.
- **No `spec.backups` field** — backups are Job-per-`Neo4jBackup`-CR only (Invariant 5). The
  `backupsPath` / chaining fields in `neo4jbackup_types.go` are CR fields, not a cluster-spec block;
  do not reintroduce a `Backups` field on the Cluster/Standalone specs.
- Validation logic does NOT live here — there are **NO webhooks** (Invariant 1). `+kubebuilder:validation:*`
  markers cover schema-level constraints; semantic validation is inline in `internal/validation/`,
  called from reconcilers. CalVer detection (`ParseVersion` / `IsCalver`) lives outside this package.
- Server-based architecture: topology specs use `servers: N` + per-server role hints, never
  `primary-*`/`secondary-*` (Invariant 5).

## Tests

Unit tests are plain `go test` (not Ginkgo) in `types_test.go` — no Kind cluster needed:

```bash
go test ./api/v1beta1/...
```

Covered by the repo-wide `make test-unit`.
